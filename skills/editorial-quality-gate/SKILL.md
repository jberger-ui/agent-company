# Editorial Quality Gate

## Purpose

Create a clear release decision that is stricter than a vague editorial review.

## Instructions

1. Check whether the angle is differentiated.
2. Confirm metadata and publish readiness.
3. Identify factual or framing risks.
4. Separate fixable issues from blocking issues.
5. Return a single decision plus follow-up actions.

## Decision enum

- approve
- approve_with_fixes
- block_pending_revision
- reject
